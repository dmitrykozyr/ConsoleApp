DevOps with AWS
Сервисы:
- CodeCommit - работа с GIT
- IAM - создание юзера, добавление его в группы и предоставление прав
- CodeBuild - сбилдить проект
https://www.linkedin.com/learning/devops-with-aws/create-first-repo?autoAdvance=true&autoSkip=false&autoplay=true&resume=false&u=2113185
В AWS сервисе 'CodeCommit' создаем репозиторий
В AWS сервисе 'IAM' на вкладке 'Users' жмем 'Add user' - создаем юзера, скачиваем .csv файл с его ключами
Заходим на страницу этого юзера на вкладку 'Security credentials', под 'HTTPS Git credentials for AWS CodeCommit' жмем 'Generate credentials', скачиваем
В AWS сервисе 'CodeCommit' справа в выпадаюзем списке выбираем 'Clone URL' - в буфер копируется URL
На компьютере открываем папку с проектом, клонируем туда проект из AWS (можно проект с кодом в папку добавить потом)
CMD: git clone https://git-codecommit.us-east-1.amazonaws.com/v1/repos/cicd-repo
Логин/пароль будут в скачанном файле, который получили после создания юзера
Добавляем в папку код и коммитим:
- git add .
- git commit -m"First commit"
- git push
В AWS сервисе 'CodeCommit' в окне 'Repositories' появится наш репозиторий
В AWS сервисе 'CodeBuild' создаем build project и заполняем поля:
- Source provider: AWS Code Commit
- Repository: имя репозитория
- Operating System: Amazon Linux 2
- Runtimes: Standard
- Image: 3.0
После создания жмем 'Start build'
В проекте есть файл buildspec.yml - в нем хранятся настройки билда

- IAM выдает права на разные сервисы и разные действия
- S3 - сервис AWS, который используется всегда
  Безопасное хранилище файлов, можно делать бекапы БД
  Все файлы в S3 хранятся в Buckets с уникальными именами, это как названия сайтов
  Например, https://uniqueNameOfBucket.s3.amazonaws.com
 	        https://uniqueNameOfBucket.eu-central-1.amazonaws.com
- EC2 - тоже используется практически везде, где используется AWS
  Это виртуальные машины (Htpervisor) - объединяют множество разных ресурсов в один большой ресурс, как с докером
- RDS - сервис для работы с БД
  У БД может быть много копий, например, 15
  Когда вставляем что-то в БД, то через некоторое время эти изменения синхронизируются с другими БД
  Это называется репликация
- DynamoDB
- Elasticache
- CloudTrail - помогает отслеживать активность пользователей
- Trusted Advisor  - сканирует наши AWS сервисы на наличие уязвимостей и места, где нужна донастройка
