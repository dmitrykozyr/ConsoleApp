Kubernetes — система оркестрации контейнеров. Она автоматически:
- запускает Docker-контейнеры на нескольких машинах
- перезапускает упавшие контейнеры
- масштабирует приложение (больше/меньше копий)
- балансирует трафик между копиями
- обновляет версии без простоя (rolling update)

┌─────────────────────────────────────────────────────────┐
│                    Kubernetes Cluster                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │   Node 1    │  │   Node 2    │  │   Node 3    │     │
│  │ ┌─────────┐ │  │ ┌─────────┐ │  │ ┌─────────┐ │     │
│  │ │  Pod    │ │  │ │  Pod    │ │  │ │  Pod    │ │     │
│  │ │(.NET app)│ │  │ │(.NET app)│ │  │ │(.NET app)│ │     │
│  │ └─────────┘ │  │ └─────────┘ │  │ └─────────┘ │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│         ▲                                                │
│         │ Service (балансировщик)                        │
│         │ Ingress (внешний доступ)                       │
└─────────────────────────────────────────────────────────┘

Объект				| Что делает
--------------------------------------------------------------------------
Pod					| Минимальная единица — один или несколько контейнеров
Deployment			| Управляет Pod'ами: сколько копий, как обновлять
Service				| Стабильный сетевой адрес для доступа к Pod'ам
Ingress				| HTTP-маршрутизация снаружи кластера
ConfigMap / Secret	| Конфигурация и секреты
Namespace			| Логическое разделение ресурсов

Файлы конфигурации:
├── deployment.yaml     # Как запускать приложение
├── service.yaml        # Как к нему обращаться
└── configmap.yaml      # Внешняя конфигурация

В Dockerfile описано:
- взять среду для сборки .NET
- скомпилировать программу
- взять облегченную среду для запуска
- положить туда скомпилированную программу
- сказать: «программа слушает порт 8080»


Команды для терминала Docker:

	Перейти в проект
		C:\Repos\ConsoleApp\Projects\Kubernetes\Kubernetes_

	Команда сборки
		docker build -t k8s-demo-app:1.0 .

	Проверка
		docker images k8s-demo-app

	Убедиться, что кластер работает
		kubectl config get-contexts
		kubectl get nodes

	Кластер не видит локальный образ из docker images  - нужно загрузить
		kind load docker-image k8s-demo-app:1.0 --name desktop

	Применить настройки Kubernetes
		kubectl apply -f k8s/

	Проверить, что копии запустились
		kubectl get pods

	Открыть доступ к приложению. Пока окно открыто — приложение доступно
		kubectl port-forward svc/k8s-demo-app-service 30080:80


Команды для другого терминала PowerShell:

	Запрос 1 — главная страница
		curl http://localhost:30080/

	Запрос 2 — балансировка (выполнить 5–10 раз подряд, должно менять хост)
		curl http://localhost:30080/

	Проверка здоровья
		curl http://localhost:30080/health

	API с данными
		curl http://localhost:30080/api/items

	Посмотреть сервисы
		kubectl get svc

	Посмотреть логи
		kubectl logs -l app=k8s-demo-app

	Увеличить число копий до 5
		kubectl scale deployment k8s-demo-app --replicas=5
		kubectl get pods

	Перезапустить все копии
		kubectl rollout restart deployment k8s-demo-app
