Различия между RabbitMq и Kafka

RabbitMQ
	- сам проталкивает сообщения потребителям (Consumers), как только они появляются
	- подходит для задач, которые нужно выполнить немедленно
	- сообщение удаляется после подтверждения успешной обработки
	- очень быстрый для простых сценариев, но сложно масштабируется на огромные потоки данных (терабайты в час)

Kafka
	- сообщение сохраняется на диске от нескольких дней до бесконечности, его можно перечитать заново
	- потребители сами приходят в Kafka и вытягивают данные, когда готовы их обработать, что позволяет потребителям работать в своем темпе
	- создана для Big Data - может переваривать миллионы сообщений в секунду, распределяя их по множеству серверов (кластеру)

	 Запустить Kafka через Docker
	 docker run -d --name kafka -p 9092:9092 apache/kafka:latest 
	 Запустить оба проекта одновременно - Producer и Consumer 
	 Отправить POST запрос на http://localhost:5000/send?message=HelloKafka 
	 В консоли Worker-сервиса увижу: ПОЛУЧЕНО: HelloKafka
