Ocelot и Yarp позволяют получить данные из микросервиса, не зная его реального адреса
Для защиты внутренней сети блокирует пути, не прописанные в конфиге и не содержащие секретный ключ,
наличие которого проверяем в middleware сервиса Vault_

Сервис Vault_ работает на
https://localhost:7291

Хочу, чтобы клиенты обращались к нему через гейтвей по адресу
https://localhost:5008

Перехожу по адресу
https://localhost:5008/vault-service/GetSecrets

Гейтвей видит совпадение и перенаправляет запрос на
https://localhost:7291/api/Vault/GetSecrets
